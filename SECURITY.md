# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do not open a public GitHub issue** for security vulnerabilities.

### How to report

1. Go to [GitHub Security Advisories](https://github.com/NGirchev/open-daimon/security/advisories/new).
2. Click **Report a vulnerability**.
3. Fill in the form with:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)
4. Submit the advisory.

Your report will be reviewed privately by the maintainers.

### Response time

We aim to acknowledge and respond to security reports within **14 days**.

### Disclosure

- Vulnerabilities will be addressed promptly.
- A fix will be developed and tested before public disclosure when possible.
- Security advisories will be published when fixes are available.
- Credit will be given to reporters unless they prefer to remain anonymous.

For more information, see [GitHub’s Security Policy](https://docs.github.com/en/code-security/security-advisories/about-github-security-advisories).
